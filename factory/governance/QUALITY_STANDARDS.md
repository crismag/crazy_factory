# Quality Standards

## Core Standard

Work is complete only when it is bounded, understandable, evidenced, reviewed, and recorded.

## Documentation Quality

- Each document has a clear purpose.
- Current state and future roadmap are distinct.
- Cross-references support navigation.
- Stable IDs are used for tasks, decisions, risks, and debt.
- Authority boundaries are explicit.
- Unknowns are recorded honestly.

## Future Implementation Quality

- Changes satisfy planned acceptance criteria.
- Scope is minimal and maintainable.
- Existing patterns are respected.
- Relevant validation passes or gaps are explicitly accepted.
- Reviewer findings are resolved or escalated.
- Local checkpoints are traceable.

## Completion Bar

No task may be marked done solely because files changed. Completion requires evidence and memory updates.
