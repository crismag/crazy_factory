# Approval Rules

## Owner Authority

The owner is the final authority for consequential changes, external actions, and restricted repository operations.

## Actions Requiring Explicit Approval

- beginning implementation after documentation bootstrap
- changing governance or authority boundaries
- accepting consequential architecture decisions
- push
- merge
- branch deletion
- enabling external integrations
- sending project data to external oversight providers
- enabling scheduled autonomous operation
- enabling automatic checkpoint commits
- enabling automatic milestone merges
- enabling multi-project operation

## Approval Evidence

Record approval in a session report, decision log, planned task, or owner instruction. Approval must be specific enough to identify the action and scope.

## No Implied Approval

Prior approval for one task or integration does not automatically authorize later unrelated work.
