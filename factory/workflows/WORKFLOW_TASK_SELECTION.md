# Workflow: Task Selection

## Purpose

Choose the next smallest valuable safe task and define it clearly enough for execution and review.

## Inputs

- backlog
- roadmap
- current state
- previous report
- decisions
- risks
- owner priorities

## Procedure

1. Filter out blocked, unapproved, and dependency-incomplete work.
2. Prefer safety and milestone blockers before optional improvements.
3. Compare value, risk, scope, and verifiability.
4. Select one bounded task.
5. State objective, scope, exclusions, criteria, validation, and stop conditions.
6. Confirm whether approval is required before execution.

## Outputs

Use [../templates/PLANNED_TASK_TEMPLATE.md](../templates/PLANNED_TASK_TEMPLATE.md).

## Failure Handling

If no task is safe and ready, return to Architect expansion, apply stall detection, or escalate to the owner.
