# Workflow: Stall Detection

## Purpose

Detect repeated activity that does not produce meaningful progress and convert it into a clear escalation or replanning event.

## Stall Signals

- the same blocker recurs across sessions
- the same task repeatedly returns from review without a new strategy
- reports repeat observations without changed evidence
- planning cycles without selecting a safe task
- validation cannot progress because a dependency remains unavailable

## Procedure

1. Compare the current session with recent reports and failure memory.
2. Identify repeated actions, blockers, and unchanged evidence.
3. Narrow the task or return to Architect expansion when possible.
4. Record what has already been tried.
5. Escalate with a concise decision request when owner input is needed.

## Exit Criteria

Resume only with a meaningfully different bounded action or an owner decision.
