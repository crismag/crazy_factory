# Workflow: Reporting

## Purpose

Produce an accurate session account for the owner and future workers.

## Procedure

1. Identify the active task, session outcome, and worker roles used.
2. Summarize work performed and files changed.
3. Record validation evidence and unavailable checks.
4. Record review outcome and repository outcome.
5. Capture decisions, failures, risks, debt, and open questions.
6. Update memory and backlog as applicable.
7. Write the next recommended action.

## Outputs

Use [../templates/SESSION_REPORT_TEMPLATE.md](../templates/SESSION_REPORT_TEMPLATE.md) or [../REPORT_TEMPLATE.md](../REPORT_TEMPLATE.md).

## Quality Rule

Reports must distinguish evidence, inference, and unresolved uncertainty.
