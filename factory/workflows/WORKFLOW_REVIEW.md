# Workflow: Review

## Purpose

Independently assess whether work is correct, scoped, maintainable, validated, and governance-compliant.

## Procedure

1. Read the planned task and exclusions.
2. Inspect changed files and repository diff.
3. Compare changes to architecture and decisions.
4. Evaluate validation evidence and gaps.
5. Identify defects, regressions, scope creep, and governance violations.
6. Classify findings as `blocking`, `major`, `minor`, or `note`.
7. Approve only when blocking findings are resolved.

## Outputs

Use [../templates/REVIEW_TEMPLATE.md](../templates/REVIEW_TEMPLATE.md).

## Escalation

Escalate governance breaches, unsafe repository state, or unresolved critical risk to the owner.
