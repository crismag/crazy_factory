# Roadmap

## Purpose

This roadmap orders future capability areas without committing to implementation details. Each milestone requires owner review before implementation begins.

## Milestone 0: Documentation Bootstrap

Define the operating system for future work: contract, lifecycle, governance, roles, workflows, templates, context records, backlog, and risk controls.

Exit criteria:

- requested documentation package exists
- documents are internally consistent
- no implementation artifacts are included
- owner approves the package for future planning

## Milestone 1: Minimal Local Apprentice

Plan a local-first prototype capable of understanding a project goal, inspecting local state, selecting one bounded task, recording a report, and preserving memory.

## Milestone 2: Local Model Integration

Plan support for Ollama and local models. Define model selection, context boundaries, observability, failure behavior, and quality expectations before implementation.

## Milestone 3: Periodic Operation

Plan cron or equivalent scheduling with explicit session boundaries, lock behavior, recovery rules, and owner controls.

## Milestone 4: Tool Integration

Evaluate MCP integration for controlled access to tools and context sources. Keep authority narrow and auditable.

## Milestone 5: Oversight

Evaluate Codex oversight and Claude oversight as optional review or planning layers. Define data handling and approval boundaries before connecting external services.

## Milestone 6: Multi-Model Collaboration

Explore specialized model assignments, conflict resolution, evidence sharing, and escalation patterns after the single-model workflow is stable.

## Milestone 7: Multi-Project Operation

Explore isolated project contexts, scheduling fairness, per-project governance, and cross-project data boundaries.
