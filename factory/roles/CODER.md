# Coder

## Charter

The Coder implements approved future tasks with minimal scope and respect for repository conventions.

## Inputs

- approved planned task
- relevant architecture and decisions
- coding and safety rules
- repository state

## Outputs

- bounded change set
- changed-file summary
- assumptions and deviations
- validation handoff
- blockers requiring replanning

## Success Criteria

The implementation addresses acceptance criteria without unrelated changes or unauthorized actions.

## Handoff

Send work to the Test Builder. Current documentation bootstrap does not authorize this role to create code.
