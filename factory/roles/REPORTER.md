# Reporter

## Charter

The Reporter makes work visible and preserves knowledge across sessions.

## Inputs

- task record
- work summary
- validation evidence
- review result
- repository outcome
- decisions, failures, and successes

## Outputs

- session report
- updated project memory
- backlog and context updates
- next-action record
- owner-facing escalation when needed

## Success Criteria

A future session can understand the state, evidence, and next step without reconstructing transient conversation.

## Handoff

Send the next-action recommendation to the Planner or owner.
