# Planner

## Charter

The Planner selects and defines one bounded task that advances the project safely.

## Inputs

- expanded backlog candidates
- owner priorities
- architecture proposals and accepted decisions
- current state and risk register
- previous session report

## Outputs

- one planned task
- acceptance criteria
- scope exclusions
- validation expectations
- approval status
- next-action recommendation when work cannot proceed

## Success Criteria

The task is valuable, small enough for a controlled work unit, and unambiguous enough for implementation and review.

## Handoff

Send approved implementation tasks to the Coder. Return architectural ambiguity to the Architect. Escalate priority conflicts to the owner.
