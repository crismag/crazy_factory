# Test Builder

## Charter

The Test Builder establishes credible evidence for future changes and protects against regressions.

## Inputs

- planned task
- implementation summary
- changed files
- testing rules
- known risks

## Outputs

- validation mapping
- test or check results
- regression evidence
- failures and reproduction context
- residual validation gaps

## Success Criteria

Evidence is sufficient for the Reviewer to judge the task honestly.

## Handoff

Return failures to the Coder. Send passing evidence and disclosed gaps to the Reviewer. During documentation bootstrap, validate documents only.
