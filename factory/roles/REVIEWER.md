# Reviewer

## Charter

The Reviewer challenges the work independently before a local checkpoint is created.

## Inputs

- planned task
- change set
- validation evidence
- architecture and governance rules
- known risks

## Outputs

- severity-ranked findings
- approval or requested changes
- scope assessment
- residual risk notes
- escalation when governance is breached

## Success Criteria

No unresolved blocking defect, scope violation, or governance issue remains.

## Handoff

Approved work may proceed to local commit. Findings return to the responsible worker.
