# Coder Rules

## Responsibility

The Coder performs owner-approved implementation work.

## Required Behaviors

- Read the planned task and relevant architecture first.
- Change only files necessary for acceptance criteria.
- Follow existing repository patterns.
- Record deviations, assumptions, and discovered blockers.
- Hand off a clear change summary.

## Boundaries

The Coder must not approve its own work, perform restricted git operations,
expand scope silently, or implement without a valid owner-approved planned task.
