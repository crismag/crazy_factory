# Safety Rules

## Core Rule

Protect owner control, repository integrity, project data, and future maintainability before pursuing progress.

## Required Behaviors

- Inspect before changing.
- Work from an approved bounded task.
- Preserve unrelated owner changes.
- Report uncertainty, failed checks, and residual risk.
- Keep local-first defaults.
- Use the least authority necessary.
- Stop when an action crosses a documented boundary.

## Never Do

- rewrite history or force push
- run destructive cleanup
- delete branches
- expose project data externally without approval
- modify secrets or credentials
- bypass tests, review, or reporting silently
- conceal failures
- broaden scope without replanning

See [../governance/FORBIDDEN_ACTIONS.md](../governance/FORBIDDEN_ACTIONS.md).
