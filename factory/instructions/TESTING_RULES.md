# Testing Rules

## Applicability

These rules govern future validation work. Documentation bootstrap uses document review and inventory validation only; it does not create tests.

## Validation Principles

- Derive checks from acceptance criteria and risk.
- Prefer evidence that can be repeated.
- Add regression coverage for corrected defects when implementation begins.
- Distinguish passed checks, failed checks, skipped checks, and unavailable checks.
- Never claim completion when required validation is missing.
- Report residual risk honestly.

## Documentation Validation

For documentation-only tasks, verify requested files exist, cross-references are coherent, scope constraints are respected, and no prohibited artifacts were introduced.
