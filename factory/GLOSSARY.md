# Glossary

| Term | Meaning |
| --- | --- |
| Apprentice | The bounded, owner-supervised operating posture of Crazy Factory |
| Approval gate | A checkpoint requiring explicit owner authorization |
| Architecture proposal | A documented option for system shape or boundaries |
| Backlog | Canonical inventory of future work |
| Bounded task | A task with explicit objective, scope, exclusions, criteria, and validation |
| Context | Durable project information loaded for a session |
| External action | An operation that publishes, transfers, or changes state outside the local repository |
| Factory | The complete Crazy Factory operating system |
| Memory | Durable records preserving knowledge across runs |
| Oversight | Optional review, planning, or supervision by an additional model or system |
| Restricted action | An operation allowed only after explicit approval |
| Session | One bounded unit of autonomous activity |
| Stall | Repeated activity without meaningful progress |
| Worker | A specialized role with a defined responsibility and handoff contract |
